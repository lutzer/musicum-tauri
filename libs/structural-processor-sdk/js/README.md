# @musicum/structural-processor-sdk

TypeScript SDK that wraps the `structural-processors` WASM module for use in the browser. Provides `StructuralProcessorManager` for applying a chain of structural audio edits and mapping times between the raw and processed audio domains.

## Overview

The manager loads `structural-processor.wasm` at runtime, applies an ordered list of structural edits to an `AudioBuffer`, and keeps a processed `AudioBuffer` in sync. It also exposes bidirectional time mapping so the playhead and any timeline markers remain accurate after edits.

## Installation

This is a private package resolved via Vite alias:

```typescript
// vite.config.ts
'@structural-processor-sdk': resolve(__dirname, '../../libs/structural-processor-sdk/js/src')
```

The frontend static folder must contain:

```
apps/frontend/static/
  structural-processor.wasm
  structural-processor-descriptors.json
```

Both files are generated by `libs/structural-processors/build.sh`.

## API

### `createStructuralProcessorManager()`

Returns a `StructuralProcessorManager` instance.

```typescript
import { createStructuralProcessorManager } from '@structural-processor-sdk';

const manager = createStructuralProcessorManager();

// Called whenever the processed audio changes
manager.onChange = (processedBuffer: AudioBuffer) => { ... };

// Apply (or update) a list of edits to raw audio
await manager.sync(edits, rawBuffer);

// Clean up WASM resources
manager.dispose();
```

### `StructuralProcessorManager`

```typescript
interface StructuralProcessorManager {
  /** Sync edits and raw audio. Triggers onChange if the output changed. */
  sync(states: StructuralProcessorState[], rawBuffer: AudioBuffer | null): Promise<void>;

  /** Look up a processor module by type id for validation / descriptor access. */
  getModule(type: string): { descriptor: StructuralProcessorDescriptor; validate(params: Record<string, number>): boolean } | undefined;

  /** Map a time in the processed domain back to the source (raw) domain. */
  mapToSourceTime(processedTime: number): number;

  /** Map a time in the source domain forward to the processed domain. */
  mapToProcessedTime(rawTime: number): number;

  /** Clamp a raw time to the nearest reachable position after all edits. */
  clampToAllowedTime(rawTime: number): number;

  /** Returns false if a raw time falls inside a removed region. */
  isTimeReachable(rawTime: number): boolean;

  /** Called with the new processed AudioBuffer whenever edits are applied. */
  onChange?: (processed: AudioBuffer) => void;

  dispose(): void;
}
```

### Types

```typescript
interface StructuralProcessorState {
  id: string;           // instance UUID (stable across re-renders)
  type: string;         // processor type: 'trim' | 'cut' | 'slice' | 'crop'
  enabled: boolean;
  parameters: Record<string, number>;
}

interface StructuralProcessorDescriptor {
  id: string;
  name: string;
  parameters: StructuralProcessorParameterDescriptor[];
}

type StructuralProcessorParameterDescriptor =
  | { type: 'time'; id: string; name: string; default: number }
  | { type: 'int';  id: string; name: string; default: number; min: number; max: number };
```

## Built-in Processors

| type | Parameters | Effect |
|---|---|---|
| `trim` | `start` (s), `end` (s) | Removes `start` seconds from the beginning and `end` seconds from the end |
| `cut` | `from` (s), `to` (s) | Removes the region `[from, to]` and concatenates the remaining audio |
| `slice` | `slices` (int), `select_slice` (int) | Divides audio into `slices` equal parts and keeps part `select_slice` (0-based) |
| `crop` | `from` (s), `to` (s) | Keeps only the region `[from, to]` |

## Time Mapping

Structural edits change the duration and layout of audio, so any time-based UI element (playhead, waveform cursor) must map between domains:

```
Source domain  ──► map_time_forward ──► Processed domain
               ◄── map_time_back    ◄──
```

```typescript
// Where does raw second 3.5 land after edits?
const playbackTime = manager.mapToProcessedTime(3.5);

// Which raw position does the user's click correspond to?
const sourceTime = manager.mapToSourceTime(clickedTime);

// Snap a raw time out of a cut region
const safe = manager.clampToAllowedTime(rawTime);
```
